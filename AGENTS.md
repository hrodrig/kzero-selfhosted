# Agent Guidelines (kzero-selfhosted)

- **`.cursor/`** is **local-only** (your machine, not in Git). Do **not** add **`!.cursor/`** to **`.gitignore`** allowlist — rules there stay between you and the editor/agent.
- Use **English** for all project artifacts (code, comments, commit messages, docs, README).
- **Scope:** **kzero-selfhosted** documents how to run **kzero** on a **bastion or automation host** (cron, systemd, CI) and ships **`run/docker/`**, **`run/examples/`**, and optional **kind** e2e under **`testing/`**. **kzero** is **not self-contained**: it requires external **`kubectl`** and (when the config uses **`release.*`** steps) **`helm`** on **`PATH`**, plus a usable **`kubeconfig`**. It does **not** ship Docker Compose, Helm charts for running **kzero** in-cluster, or in-cluster Kubernetes workloads — the CLI runs on **hosts** that can reach the API server, not as a substitute for **`kubectl`** inside the cluster.
- **[hrodrig/kzero](https://github.com/hrodrig/kzero)** owns the Go CLI, application **`VERSION`**, tests, **`make release-check`**, and **`ghcr.io/hrodrig/kzero`** images.
- Follow **git flow** when this repo uses branches: work on **`develop`**; **`main`** for reviewed snapshots; annotated tags **`v<semver>`** on **`main`** for releases of **this** repo (see root **`VERSION`**).
- **`VERSION`** (repository root): **kzero-selfhosted** semver (`0.1.6` style, no **`v`**). When it changes, update the static **Version** badge URL in **`README.md`** to match and record user-facing changes in **`CHANGELOG.md`** (Keep a Changelog style).
- **Upstream pin for e2e/docs:** kind smoke and Docker examples assume **kzero** [v0.6.1+](https://github.com/hrodrig/kzero/releases/tag/v0.6.1) (notify, verify, probe, preflight, man page, Homebrew, plus **`run.execution`**, server-side dry-run, live **retry**, sequential engine).
- **Optional e2e:** **`make test-kind-e2e`** (see **`testing/kind/README.md`**) — disposable **kind** cluster + **kzero** live **down**/**up**; requires **kzero** on **`PATH`** (or **`KZERO_BIN`**).
- **Example YAML** under **`run/examples/`** must stay consistent with **[kzero specifications](https://github.com/hrodrig/kzero/blob/main/docs/SPECIFICATIONS.md)** (`schema_version`, pipelines, etc.).
- Do not commit without first showing the proposed commit message and getting **explicit user approval** (same convention as **kzero**).
